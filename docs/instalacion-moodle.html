Guia para instalar la libreria UA en Moodle

Documento de apoyo para pegar las fuentes, el CSS y el JS de la libreria en los campos de HTML adicional de Moodle.

Antes de empezar

Usa esta guia en un curso o entorno de prueba antes de activar la libreria para toda la plataforma.

Checklist inicial
Ruta habitual

Administracion del sitio > Apariencia > HTML adicional.

Campos que se usan

additionalhtmlhead para fuentes y CSS. additionalhtmlfooter para JavaScript.

Raw CSS de origen

https://raw.githubusercontent.com/MateoVR13/lib-uamerica/refs/heads/main/assets/css/uamerica-moodle.css

Raw JS de origen

https://raw.githubusercontent.com/MateoVR13/lib-uamerica/refs/heads/main/assets/js/uamerica-moodle.js

Si Moodle esta en ingles, los campos aparecen como Within HEAD y Before BODY is closed.
Para Moodle se pegan los enlaces CDN de jsDelivr, no los raw.githubusercontent.com. La conversion correcta es raw.githubusercontent.com/MateoVR13/lib-uamerica/refs/heads/main/... a cdn.jsdelivr.net/gh/MateoVR13/lib-uamerica@main/....

1. Within HEAD

Pega este bloque en additionalhtmlhead. Carga fuentes y CSS.

additionalhtmlhead

Codigo para HEAD

En este campo no pegues el JavaScript. El JS debe ir antes de cerrar el BODY para evitar bloqueos y para que los componentes ya existan cuando se inicialicen.

2. Before BODY is closed

Pega este bloque en additionalhtmlfooter. Carga la interaccion de componentes.

additionalhtmlfooter

Codigo para footer

Si algun dia decides cargar este script en el HEAD, usa defer. En additionalhtmlfooter no es obligatorio porque el script ya queda al final de la pagina.

3. Guardar y limpiar cache

Despues de guardar los cambios en Moodle, limpia cache y prueba un componente.

verificacion
  1. Guarda los cambios del formulario de HTML adicional.
  2. Limpia cache de Moodle desde administracion del sitio si tienes permisos.
  3. Abre un curso de prueba y pega un componente generado por el personalizador.
  4. Comprueba que estilos, fuentes, carruseles, tabs, acordeones y audios funcionen.

Componente minimo de prueba

4. Actualizaciones y cache de jsDelivr

Cuando subas cambios al repositorio, puede ser necesario purgar CDN y limpiar cache de Moodle.

cache
Purgar CSS

https://purge.jsdelivr.net/gh/MateoVR13/lib-uamerica@main/assets/css/uamerica-moodle.css

Purgar JS

https://purge.jsdelivr.net/gh/MateoVR13/lib-uamerica@main/assets/js/uamerica-moodle.js

Para desarrollo puedes usar @main. Para produccion es mejor crear un tag, por ejemplo @v1.0.0, y apuntar Moodle a esa version fija.

5. Regla para pegar componentes

Todo componente generado debe ir dentro de .ua-moodle.
